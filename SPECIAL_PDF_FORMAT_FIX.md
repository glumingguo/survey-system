# PDF导入功能修复 - 支持特殊PDF格式

## 问题描述

用户上传的示例PDF文件（"沈阳27白领人妻戚睿入门篇.pdf"）无法正确解析，显示"PDF解析失败"。

## 根本原因分析

经过详细分析，发现用户的PDF文件使用了**非常特殊的格式**，与标准的问卷格式完全不同：

### 用户PDF的格式特点

1. **问题和编号分离**
   - 问题标题在单独一行
   - 问题编号在单独一行（例如："3. *"）
   - 问题和答案混在一起

2. **示例结构**：
```
你的昵称或者名字       <-- 问题标题
3. *                   <-- 问题编号
戚睿                   <-- 答案/选项
```

```
你的年龄               <-- 问题标题
4. *                   <-- 问题编号
27                     <-- 答案/选项
```

```
面对自身的淫贱欲望你愿意把它展现出来吗   <-- 问题标题
9. *                                       <-- 问题编号
是的淫贱的我才是最真实的我                <-- 答案/选项
我一直都是这么淫贱                        <-- 答案/选项
我想试试给自己一个释放欲望的机会          <-- 答案/选项
```

3. **选项格式**
   - 没有标准的A、B、C、D选项
   - 使用中文词汇开头：是、否、好的、没有、有过等
   - 选项紧接在问题编号之后

4. **内容特点**
   - 30页，122个问题
   - 包含大量个人隐私内容
   - 混合了问题和已填写的答案

## 解决方案

### 1. 新增特殊格式识别

添加了针对这种特殊格式的识别逻辑：

```typescript
// 特殊格式：问题编号在单独一行，带有 * 标记
const specialNumberPattern = /^(\d+)\.\s*\*$/;  // 例如：3. *
```

### 2. 待处理标题机制

实现了`pendingQuestionTitle`机制：
- 当发现潜在的问题标题时，先存储起来
- 遇到问题编号行（如"3. *"）时，将存储的标题与编号关联

```typescript
let pendingQuestionTitle: string | null = null;

// 检查是否是潜在的问题标题
if (!currentQuestion && !pendingQuestionTitle) {
  pendingQuestionTitle = line;
  continue;
}

// 遇到编号时使用存储的标题
if (specialMatch) {
  const title = pendingQuestionTitle || `问题 ${questionNumber}`;
  // ...
  pendingQuestionTitle = null;
}
```

### 3. 智能中文选项识别

添加了对常见中文选项词的识别：

```typescript
// 检查常见的中文选项词
if (!isOption && (line.startsWith('是') || line.startsWith('否') ||
                  line.startsWith('没有') || line.startsWith('有过') ||
                  line.startsWith('好的') || line.startsWith('我'))) {
  // 视为选项
  isOption = true;
  optionText = line;
}
```

### 4. 改进的过滤逻辑

- 过滤页码（`Page 1`、`=== Page N ===`等）
- 过滤短行（长度 < 5）
- 过滤纯数字行
- 避免重复选项

## 兼容性

### 支持的格式

现在支持以下所有格式：

#### 标准格式
1. `1. 问题内容`
2. `1、问题内容`
3. `1．问题内容`
4. `(1) 问题内容`
5. `一. 问题内容`
6. `Q1: 问题内容`
7. `第1题：问题内容`
8. `【1】问题内容`

#### 特殊格式（新增）
9. `问题标题`（单独一行）<br>`3. *`（下一行）

#### 选项格式
1. `A. 选项`
2. `(A) 选项`
3. `① 选项`
4. `1. 选项`
5. `是`、`否`、`好的`、`没有`、`有过`等中文选项（新增）

## 技术细节

### 核心算法

```
for each line:
  if line matches "数字. *" pattern:
    create question with pending title
  else if line looks like question title:
    store as pending title
  else if line looks like option:
    add to current question
  else:
    add as description
```

### 关键改进

1. **两阶段识别**：
   - 第一阶段：识别潜在问题标题
   - 第二阶段：遇到编号时创建问题

2. **模糊匹配**：
   - 不要求严格的格式
   - 允许多种变体

3. **智能过滤**：
   - 自动排除页码、分隔符
   - 避免重复内容

## 已知限制

1. **混合内容**：
   - PDF中包含已填写的答案
   - 答案会被识别为选项
   - 需要手动清理

2. **格式不一致**：
   - 不同PDF可能使用不同格式
   - 需要针对特定格式优化

3. **中文编码**：
   - 部分PDF可能有编码问题
   - 需要良好的UTF-8支持
